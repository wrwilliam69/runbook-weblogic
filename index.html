⚠️ Importante — Security Group

Abrir solo puerto 22 (SSH) durante la instalación. No expongas consola de administración (7001, 7002) hasta terminar la instalación.

2 Script User Data (arranque automático)
ℹ️ ¿Qué es User Data?

Es un script que EC2 ejecuta una sola vez al crear la instancia. Evita instalar herramientas manualmente después de conectarte.

user-data.sh
#!/bin/bash
# Actualizar todos los paquetes del sistema
dnf update -y

# Instalar herramientas necesarias:
#   lvm2    → gestión de volúmenes lógicos (LVM)
#   sysstat → iostat, sar para monitoreo de disco/CPU
#   lsof    → ver qué procesos tienen archivos abiertos
#   unzip   → descomprimir archivos ZIP
dnf install -y lvm2 sysstat lsof unzip
Paso 2
LVM — Volúmenes Lógicos
ℹ️ ¿Qué es LVM?

LVM (Logical Volume Manager) — gestor de volúmenes lógicos del kernel Linux. Permite combinar múltiples discos físicos en un solo espacio flexible y redimensionable sin reiniciar.

  • PV (Physical Volume) = disco físico preparado para LVM
  • VG (Volume Group) = agrupación de uno o más PV
  • LV (Logical Volume) = partición virtual sobre el VG, donde montas el filesystem
1 Ver discos disponibles
Terminal
# Listar bloques de disco (árbol visual de discos y particiones)
lsblk
✅ Salida esperada

Verás nvme0n1 (root), nvme1n1 (LAB2) y nvme2n1 (LAB3) sin montar.

2 Crear Physical Volumes
# pvcreate marca discos para que LVM los reconozca
pvcreate /dev/nvme1n1 /dev/nvme2n1
3 Crear Volume Group
# vgcreate nombre_vg disco1 disco2
# Une nvme1n1 (9GB) + nvme2n1 (15GB) = 24GB usables
vgcreate vg_weblogic /dev/nvme1n1 /dev/nvme2n1
4 Crear Logical Volumes
# lv_domain: donde vive el dominio WebLogic (middleware)
lvcreate -n lv_domain -L 12G vg_weblogic

# lv_log: donde se guardarán los logs de la aplicación
lvcreate -n lv_log -L 8G vg_weblogic
ℹ️ Flags explicados sin símbolos matemáticos
  • -n nombre → asigna un nombre al volumen lógico
  • -L 12G → tamaño exacto: L mayúscula igual a 12 gigabytes
Paso 3
Filesystem — Formato, Montaje y Persistencia
1 Formatear con XFS
ℹ️ ¿Qué es XFS?

Sistema de archivos de alto rendimiento recomendado en RHEL. Soporta archivos grandes y tiene buen rendimiento de I/O para bases de datos y middleware.

# mkfs.xfs = "make filesystem de tipo XFS"
mkfs.xfs /dev/vg_weblogic/lv_domain
mkfs.xfs /dev/vg_weblogic/lv_log
2 Crear directorios de montaje
# /u01: convención Oracle para instalaciones (Unidad 01)
mkdir /u01

# /logs: centraliza todos los logs del servidor
mkdir /logs
3 Montar los volúmenes
mount /dev/vg_weblogic/lv_domain /u01
mount /dev/vg_weblogic/lv_log    /logs
4 Persistencia con /etc/fstab
⚠️ Por qué necesitas fstab

Sin esta configuración, al reiniciar la instancia los volúmenes quedan sin montar y WebLogic no arranca. fstab monta discos automáticamente al boot.

# Obtener UUID de los volúmenes (identificador único del filesystem)
blkid | grep vg_weblogic
# Editar fstab con vi
vi /etc/fstab

# Agregar al final del archivo (sustituir xxxxx por UUID real):
# FORMATO: UUID=id  punto_de_montaje  tipo  opciones  dump  pass
UUID=xxxxx  /u01   xfs  defaults  0  0
UUID=xxxxx  /logs  xfs  defaults  0  0
ℹ️ Columnas de fstab
  • UUID = identificador del volumen (más estable que /dev/sdX)
  • punto de montaje = directorio donde aparecerá el disco
  • xfs = tipo de filesystem
  • defaults = opciones estándar (lectura/escritura, no-exec para swap)
  • 0 0 = sin backup automático, sin verificación fsck al boot
5 Crear estructura Oracle
# -p = crea todos los directorios intermedios que falten
mkdir -p /u01/app/oracle
mkdir -p /u01/domains
Paso 4
Swap — Memoria Virtual para WebLogic
⚠️ Error frecuente sin swap

El instalador de WebLogic verifica que exista swap mayor a 512 MB antes de continuar. Si no hay swap, lanza el error:
swap must be greater than 512MB

ℹ️ ¿Qué es Swap?

Espacio en disco que el SO usa como extensión de la RAM cuando la memoria física se llena. En producción se configura en un disco SSD dedicado; en laboratorio usamos un archivo swap (swapfile).

1 Crear y activar swapfile de 1 GB
# fallocate reserva 1 gigabyte de espacio contiguo en disco
# -l 1G = longitud igual a 1 gigabyte
sudo fallocate -l 1G /swapfile

# Permisos 600 = solo root puede leer/escribir (obligatorio para swap)
# chmod 600 = modo: solo propietario tiene lectura y escritura
sudo chmod 600 /swapfile

# Formatear el archivo como área de swap
sudo mkswap /swapfile

# Activar el swap inmediatamente
sudo swapon /swapfile

# Verificar que aparece el swap activo
free -h
✅ Salida esperada de free -h

Verás una fila Swap con total cercano a 1.0G y usado en 0B (sin usar aún).

Paso 5
Instalación WebLogic — Silent Mode
1 Subir instaladores vía SFTP (Termius)
ArchivoDescripciónDestino en servidor
fmw_14.1.2.0.0_wls.jar Instalador WebLogic /u01/
jdk-17_linux-x64_bin.tar.gz Oracle JDK 17 /u01/
2 Instalar JDK 17
# tar = herramienta de archivos comprimidos
# -x = extraer, -z = descomprimir gzip, -f = nombre de archivo
tar -xzf /u01/jdk-17_linux-x64_bin.tar.gz
3 Crear Response File
ℹ️ ¿Qué es un Response File?

Archivo de texto que responde automáticamente todas las preguntas del instalador gráfico. Permite instalar en modo silent sin GUI, ideal para servidores sin escritorio.

# cat > archivo crea el archivo con el contenido entre EOF...EOF
cat > /u01/wls.rsp << 'EOF'
[ENGINE]
Response File Version=1.0.0.0

[GENERIC]
# Ruta donde se instalará WebLogic (ORACLE_HOME)
ORACLE_HOME=/u01/app/oracle/middleware
INSTALL_TYPE=WebLogic Server
# No enviar info de seguridad a Oracle durante instalación
DECLINE_SECURITY_UPDATES=true
EOF
4 Crear Oracle Inventory
⚠️ Primera instalación Oracle en el servidor

Oracle necesita un directorio de inventario central para registrar todos sus productos instalados. Sin esto el instalador falla con error de permisos.

sudo /tmp/createCentralInventory*.sh /u01/app/oraInventory ec2-user
5 Resolver error de espacio en /tmp
⚠️ /tmp demasiado pequeño

WebLogic extrae archivos temporales en /tmp. En t2.micro el espacio es insuficiente. La solución es redirigir TMPDIR a /u01 que tiene más espacio.

# export = crea/modifica variable de entorno para la sesión actual
# TMPDIR = le dice a WebLogic dónde guardar sus temporales
export TMPDIR=/u01
6 Ejecutar instalación Silent
# Usar el JDK 17 recién extraído (no el sistema)
# -jar = ejecutar un archivo JAR como programa
# -silent = sin interfaz gráfica
# -responseFile = usar nuestro archivo de respuestas
# -invPtrLoc = ubicación del inventory Oracle
/u01/jdk-17*/bin/java -jar /u01/fmw_14.1.2.0.0_wls.jar \
  -silent \
  -responseFile /u01/wls.rsp \
  -invPtrLoc /etc/oraInst.loc
✅ Resultado esperado

installation completed successfully

Referencia
Glosario Técnico
LVM
Logical Volume Manager — Capa de abstracción entre discos físicos y el sistema de archivos. Permite crear, redimensionar y mover volúmenes sin desmontar discos ni reiniciar.
PV
Physical Volume — Un disco o partición inicializada para LVM. Analogía: ladrillo crudo.
VG
Volume Group — Agrupación de uno o más PVs. Analogía: pared construida con los ladrillos PV.
LV
Logical Volume — Partición virtual creada desde el VG. Analogía: habitación dentro de la pared.
XFS
Sistema de archivos de alto rendimiento para cargas de trabajo intensivas. Predeterminado en RHEL desde versión 7. Soporta archivos de hasta 8 exabytes.
Swap
Espacio en disco usado como extensión de RAM. El kernel mueve páginas de memoria inactivas al swap para liberar RAM física.
fstab
/etc/fstab — Tabla de sistemas de archivos. Define qué discos se montan automáticamente al iniciar el sistema operativo.
UUID
Universally Unique Identifier — Identificador único asignado a cada filesystem al formatearlo. Más confiable que /dev/sdX porque no cambia al reiniciar.
Silent Install
Modo de instalación sin interfaz gráfica, dirigido por un archivo de respuestas (.rsp). Fundamental en servidores sin entorno de escritorio.
ORACLE_HOME
Variable de entorno y directorio donde se instala el software Oracle (WebLogic, DB, etc.). Por convención suele ser /u01/app/oracle/middleware.
User Data (EC2)
Script que Amazon EC2 ejecuta automáticamente durante el primer arranque de la instancia. Sirve para instalar software o configurar el SO sin intervención manual.
TMPDIR
Variable de entorno que le dice a las aplicaciones dónde crear archivos temporales. Por defecto apunta a /tmp; se puede redirigir a un disco con más espacio.
blkid
Comando que muestra el UUID y tipo de cada dispositivo de bloque (disco/volumen) del sistema. Se usa para configurar fstab.
fallocate
Reserva espacio en disco de forma instantánea sin escribir ceros. Más rápido que dd para crear swapfiles.
Memoria Técnica
Reglas Mnemotécnicas
🧠 Orden LVM — "PVerde, VGrande, LVive"

PV verde → pvcreate (prepara el disco)
VG grande → vgcreate (agrupa los discos)
LV vive → lvcreate (crea la partición virtual)

Recuerda: Primero Physical, luego Virtual Group, luego Logical.

🧠 SWAP — "FALTO CHMOD MKSWAP SWAPON"

FALlocate → reserva espacio
CHmod 600 → asegura permisos
MKswap → formatea como swap
SWAPon → activa

Frase: "Fal-Ch-Mk-On" — como encender un motor en 4 pasos.

🧠 Errores WebLogic — "JDST–Display"

JDK → Necesitas Oracle JDK 17, no OpenJDK
Disco /tmp → export TMPDIR antes de instalar
Swap → mayor a 512 MB siempre
Tinventory → crear Oracle Inventory primero
Display → usar -silent si no hay pantalla

🧠 Comandos de diagnóstico — "BDF + LS + FREE"

Blkid → ver UUID de discos
Df -h → espacio en disco montado
Free -h → RAM y Swap
LSblk → árbol de discos
LSof → archivos abiertos por proceso

Ante cualquier duda en infraestructura, ejecuta estos 5 antes de escalar.

🧠 fstab — columnas en orden

UUID → identificador del volumen
Mount point → /u01 o /logs
Tipo → xfs
Opciones → defaults
Dump → 0 (sin backup)
Pass → 0 (sin fsck)

Nemónico: "UUID Monta Tipo, Options, Dump Pass"
